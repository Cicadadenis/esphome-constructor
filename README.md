Современный визуальный конструктор для быстрого создания ESPHome YAML-конфигов под Home Assistant без ручной возни с шаблонами, шинами и GPIO.

Проект сделан как один автономный HTML-файл, который открывается прямо в браузере и сразу готов к работе.

## Что умеет

- Автоматически включает I2C, UART и OneWire по выбранным устройствам
- Управляет GPIO как умный помощник: показывает занятые, опасные и конфликтующие пины
- Не использует boot-пины ESP8266 в авто-подборе
- Поддерживает датчики и модули с 1, 2 и 3 GPIO
- Генерирует готовый YAML для ESPHome
- Позволяет включать или отключать OTA и MQTT
- Показывает визуальную распиновку выбранной платы
- Даёт ссылку на подходящий USB-UART драйвер для выбранной платы
- Поддерживает светлую и тёмную тему
- Имеет систему плагинов датчиков с пользовательскими YAML-шаблонами
- Показывает предупреждения в красивом модальном окне вместо стандартных alert

## Ключевые фишки

### Авто-конфигурация шин

Если выбранный датчик требует I2C, UART или OneWire, конструктор сам добавит нужный блок в YAML и подставит дефолтные пины для выбранной платы.

### Умный GPIO-менеджер

Конструктор:

- отслеживает занятые пины
- запрещает дублирование в одном устройстве
- предупреждает о рискованных GPIO
- показывает конфликты с шинами
- умеет автоматически подбирать безопасные пины

### Визуальная распиновка

При выборе платы рядом отображается её pinout и ссылка на драйвер для USB-UART чипа.

### Плагины датчиков

Можно добавить собственный тип устройства через YAML-шаблон и использовать переменные:

- `{{name}}`
- `{{pin}}`
- `{{pin1}}`
- `{{pin2}}`
- `{{pin3}}`
- `{{pwm_platform}}`

## Быстрый старт

1. Откройте файл `esphome-constryctor.html` в браузере.
2. Введите имя устройства, Wi-Fi и при необходимости OTA или MQTT.
3. Выберите плату.
4. Добавьте нужные датчики и модули.
5. При необходимости нажмите Авто-пин.
6. Сгенерируйте YAML.
7. Скопируйте или скачайте конфиг.
8. При желании откройте ESPHome Web для прошивки.

## Ограничения

- Проект работает как фронтенд-конструктор и не компилирует `.bin` прямо в браузере
- Для реальной сборки firmware нужен ESPHome локально или внешний backend
- Совместимость устройств сейчас оценивается в основном по доступным безопасным GPIO и типу шин
